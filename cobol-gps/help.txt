Compilador GnuCOBOL para muchos dialectos COBOL con lotes de extensiones

Modo de empleo: cobc [opciones]... fichero...

Opciones:
  -h, -help             enseña esta ayuda y sale
  -V, -version          enseña la versión del compilador y sale
  -i, -info             enseña información del compilador (compliar/entorno)
                        y sale
  -v, -verbose          verbose mode, display additional information;
                        multiple -v options increase the verbosity,
                        the maximum is 3 as follows:
                        (1) display compiler version and the commands
                        invoked by the compiler,
                        (2) pass verbose option to assembler/compiler
                        (3) pass verbose option to linker
  -q, -brief            pantallas reducidas, órdenes invocadas no mostradas
  -###                  like -v but commands not executed
  -x                    compila un programa ejecutable
 -m                     compila un módulo de carga dinámicamente (predet.)
  -j [<args>], -job[=<args>]	ejecuta un programa tras compilación, pasando <args>
  -std=<dialect>        avisos/características para un dialecto escecífico
                        <dialect> puede ser uno de:
                        predeterminado, cobol2014, cobol2002, cobol85,
                        xopen, ibm-strict, ibm, mvs-strict, mvs,
                        mf-strict, mf, bs2000-strict, bs2000,
                        acu-strict, acu, rm-strict, rm;
                        vea ficheros de configuración en el directorio de
                        configuración
  -F, -free             emplear formato de origen libre
  -fixed                emplear formato de origen fijo (predeterminado)
  -O, -O2, -O3, -Os     optimización activada
  -O0                   disable optimization
  -g                    enable C compiler debug and stack check
  -d, -debug            enable all run-time error checking,
                        equal to -fstack-check -fec=EC-ALL
  -fec=<exception-name>	enable code generation for <exception-name>,
                        sets -fsource-location
  -fno-ec=<exception-name>	disable code generation for <exception-name>
  -o <file>             lugar de salida al <fichero>
  -b                    combinar todos los ficheros de entrada en un solo
                        módulo dinámicamente cargable
  -E                    preprocesar solamente; no compilar o enlazar
  -C                    traducción solamente; convertir COBOL a C
  -S                    compilar solamente; salida de fichero ensamblado
  -c                    compila y ensambla, pero no enlaza
  -T <file>             genera y ubica un programa ancho listando dentro de <file>
  -t <file>		genera y ubica un programa listando dentro de <file>
  --tlines=<lines>      líneas especificas por página en lista, predeterminada = 55
  -P(=<dir o file>)    genera listado de programa preprocesado (.lst)
  -Xref                 genera referencia cruzada mediante 'cobxref'
                        (V. Coen's 'cobxref' debe estar en el camino)
  -I <directory>        añade <directory> para copiar/incluir ruta de búsqueda
  -L <directory>        añade el <directorio> a ruta de búsqueda de biblioteca
  -l <lib>              enlaza la biblioteca <lib>
  -A  <options>         añade <opciones> a la fase de compilador C
  -Q <options>          añade <opciones> a la frase de enlace C
  -D <define>           define <define> para compilación COBOL
  -K <entry>            generar CALL a <entry> como estática
  -conf=<file>          configuración de dialecto definido por usuario; vea -std
  -list-reserved        enseña palabras reservadas
  -list-intrinsics      enseña funciones intrínsecas
  -list-mnemonics       enseña nombres mnemónicos
  -list-system          enseña rutinas del sistema
  -save-temps[=<dir>]   save intermediate files
                        * default: current directory
  -ext <extension>      añade extensión de fichero para resolver COPY

Warning options:
  -Wall                 activar la mayoría de las advertencias (todas excepto como se anotaron debajo)
  -Wextra               like -Wall but enable some extra warning flags
  -w                    disable all warnings
  -Wno-<warning>        disable warning enabled by default, -Wall or -Wextra
  -Wadditional          additional warnings only raised with -Wall
  -Wno-unfinished       do not warn if unfinished features are used
                        * ALWAYS active
  -Wno-pending          do not warn if pending features are mentioned
                        * ALWAYS active
  -Wno-ignored-error    do not warn about errors in unreachable code parts and so normally ignored
                        * ALWAYS active
  -Wobsolete            warn if obsolete features are used
  -Warchaic             warn if archaic features are used
  -Wredefinition        warn about non-referenced ambiguous data items
  -Wtruncate            warn about field truncation from constant assignments
  -Wpossible-truncate   warn about possible field truncation
                        * NOT set with -Wall
  -Woverlap             warn about overlapping MOVE of items
  -Wpossible-overlap    warn about MOVE of items that may overlap depending on variables
                        * NOT set with -Wall
  -Wparentheses         warn about lack of parentheses around AND within OR
  -Wstrict-typing       warn strictly about type mismatch
  -Wimplicit-define     warn about implicitly defined data items
                        * NOT set with -Wall
  -Wno-corresponding    do not warn about CORRESPONDING with no matching items
                        * ALWAYS active
  -Winitial-value       warn if initial VALUE clause is ignored
  -Wprototypes          warn about missing FUNCTION prototypes/definitions
  -Warithmetic-osvs     warn if arithmetic expression precision has changed
  -Wcall-params         warn about non 01/77 items for CALL parameters
                        * NOT set with -Wall
  -Wconstant-expression  warn about expressions that always resolve to true/false
  -Wcolumn-overflow     warn about text after program-text area, FIXED format
                        * NOT set with -Wall
  -Wterminator          warn about lack of scope terminator END-XXX
                        * NOT set with -Wall
  -Wlinkage             warn about dangling LINKAGE items
                        * NOT set with -Wall
  -Wunreachable         warn about likely unreachable statements
                        * NOT set with -Wall
  -Wno-dialect          do not warn about dialect specific issues
                        * ALWAYS active
  -Wdangling-text       warn source text after program-area
                        * NOT set with -Wall
  -Wno-others           do not warn about different issues
                        * ALWAYS active
  -Werror               trata todas las advertencias como errores
  -Wno-error            don't treat warnings as errors
  -Werror=<advertencia>     tratar especificado <advertencia> como error
  -Wno-error=<warning>  don't treat specified <warning> as error

Compiler options:
  -fsign=[ASCII|EBCDIC]	define display sign representation
                        * default: machine native
  -ffold-copy=[UPPER|LOWER]	fold COPY subject to value
                        * default: no transformation
  -ffold-call=[UPPER|LOWER]	fold PROGRAM-ID, CALL, CANCEL subject to value
                        * default: no transformation
  -fdefaultbyte=<value>	initialize fields without VALUE to value
                        * decimal 0..255 or any quoted character
                        * default: initialize to picture
  -fmax-errors=<number>	maximum number of errors to report before
                        compilation is aborted
                        * default: 128
  -fintrinsics=[ALL|intrinsic function name(,name,...)]
                        intrinsics to be used without FUNCTION keyword
  -fdump=<scope>        dump data fields on abort, <scope> may be
                        a combination of: ALL, WS, LS, RD, FD, SC, LO
  -fcallfh=<name>       specifies <name> to be used for I/O
                        as external provided EXTFH interface module
  -fno-remove-unreachable	disable remove of unreachable code
                        * turned off by -g
  -ftrace               generate trace code
                        * scope: executed SECTION/PARAGRAPH
  -ftraceall            generate trace code
                        * scope: executed SECTION/PARAGRAPH/STATEMENTS
  -fsyntax-only         syntax error checking only; don't emit any output
  -fdebugging-line      enable debugging lines
                        * 'D' in indicator column or floating >>D
  -fsource-location     generate source location code
                        * turned on by -debug/-g/-ftraceall/-fec
  -fimplicit-init       automatic initialization of the COBOL runtime system
  -fno-recursive-check  disable check of recursive program call;
                        effectively compiling as RECURSIVE program
  -fstack-check         PERFORM stack checking
                        * turned on by -debug or -g
  -fwrite-after         use AFTER 1 for WRITE of LINE SEQUENTIAL
                        * default: BEFORE 1
  -fmfcomment           '*' or '/' in column 1 treated as comment
                        * FIXED format only
  -facucomment          '$' in indicator area treated as '*',
                        '|' treated as floating comment
  -fnotrunc             allow numeric field overflow
                        * non-ANSI behaviour
  -fodoslide            adjust items following OCCURS DEPENDING
                        * implies -fcomplex-odo
  -fsingle-quote        use a single quote (apostrophe) for QUOTE
                        * default: double quote
  -foptional-file       treat all files as OPTIONAL
                        * unless NOT OPTIONAL specified
  -fstatic-call         output static function calls for the CALL statement
  -fno-gen-c-decl-static-call	disable generation of C function declations
                        for subroutines with static CALL
  -fgen-c-line-directives	generate source location directives in C code;
                        * turned on by -g
  -fgen-c-labels        generate extra labels in C sources;
                        * turned on by -g
  -fno-theaders         suppress all headers and output of compilation
                        options from listing while keeping page breaks
  -fno-tsource          suppress source from listing
  -fno-tmessages        suppress warning and error summary from listing
  -ftsymbols            specify symbols in listing
  -fno-diagnostics-show-option	suppress output of option that directly
                        controls the diagnostic

Compiler dialect configuration options:
  -freserved-words=<valor>	use of complete/fixed reserved words
  -ftab-width=1..12      set number of spaces that are assumed for tabs
  -ftext-column=72..255  set right margin for source (fixed format only)
  -fpic-length=<número>	maximum number of characters allowed in the PICTURE character-string
  -fword-length=1..63    maximum word-length for COBOL (= programmer defined) words
  -fliteral-length=<número>	maximum literal size in general
  -fnumeric-literal-length=1..38	maximum numeric literal size
  -fbinary-size=<valor>  binary byte size - defines the allocated bytes according to PIC, may be one of: 2-4-8, 1-2-4-8, 1--8
  -fbinary-byteorder=<valor>	binary byte order, may be one of: native, big-endian
  -fassign-clause=<valor>	how to interpret 'ASSIGN word': as 'ASSIGN EXTERNAL word' or 'ASSIGN DYNAMIC word', may be one of: dynamic, external, ibm (= external), mf (= dynamic)
  -fscreen-section-rules=<valor>	which compiler's rules to apply to SCREEN SECTION item clauses, may be one of: acu, gc, mf, rm, std, xopen
  -fdpc-in-data=<valor>  whether DECIMAL-POINT IS COMMA has effect in XML/JSON GENERATE, may be one of: none, xml, json, all
  -ffilename-mapping     resolve file names at run time using environment variables
  -fpretty-display       alternate formatting of numeric fields
  -fbinary-truncate      numeric truncation according to ANSI
  -fcomplex-odo          allow complex OCCURS DEPENDING ON
  -findirect-redefines   allow REDEFINES to other than last equal level number
  -flarger-redefines-ok  allow larger REDEFINES items
  -frelax-syntax-checks  allow certain syntax variations (e.g. REDEFINES position)
  -fref-mod-zero-length  allow zero length reference-modification (only changed with EC-BOUND-REF-MOD active)
  -frelax-level-hierarchy	allow non-matching level numbers
  -fselect-working       require ASSIGN USING items to be in WORKING-STORAGE
  -flocal-implies-recursive	LOCAL-STORAGE SECTION implies RECURSIVE attribute
  -fsticky-linkage       LINKAGE SECTION items remain allocated between invocations
  -fmove-ibm             MOVE operates as on IBM (left to right, byte by byte)
  -fperform-osvs         exit point of any currently executing perform is recognized if reached
  -farithmetic-osvs      limit precision in intermediate results to precision of final result (less accurate)
  -fconstant-folding     evaluate constant expressions at compile time
  -fhostsign             allow hexadecimal value 'F' for NUMERIC test of signed PACKED DECIMAL field
  -fprogram-name-redefinition	program names don't lead to a reserved identifier
  -faccept-update        set WITH UPDATE clause as default for ACCEPT dest-item, instead of WITH NO UPDATE
  -faccept-auto          set WITH AUTO clause as default for ACCEPT dest-item, instead of WITH TAB
  -fconsole-is-crt       assume CONSOLE IS CRT if not set otherwise
  -fno-echo-means-secure	NO-ECHO hides input with asterisks like SECURE
  -fline-col-zero-default	assume a field DISPLAY starts at LINE 0 COL 0 (i.e. at the cursor), not LINE 1 COL 1
  -fdisplay-special-fig-consts	special behaviour of DISPLAY SPACE/ALL X'01'/ALL X'02'/ALL X'07'
  -fbinary-comp-1        COMP-1 is a 16-bit signed integer
  -fnumeric-pointer      POINTER is a 64-bit unsigned integer
  -fmove-non-numeric-lit-to-numeric-is-zero	imply zero in move of non-numeric literal to numeric items
  -fimplicit-assign-dynamic-var	implicitly define a variable if an ASSIGN DYNAMIC does not match any data item
  -fcomment-paragraphs=<compatible>	comment paragraphs in IDENTIFICATION DIVISION (AUTHOR, DATE-WRITTEN, ...)
  -fmemory-size-clause=<compatible>	MEMORY-SIZE clause
  -fmultiple-file-tape-clause=<compatible>	MULTIPLE-FILE-TAPE clause
  -flabel-records-clause=<compatible>	LABEL-RECORDS clause
  -fvalue-of-clause=<compatible>	VALUE-OF clause
  -fdata-records-clause=<compatible>	DATA-RECORDS clause
  -ftop-level-occurs-clause=<compatible>	OCCURS clause on top-level
  -fsame-as-clause=<compatible>	SAME AS clause
  -ftype-to-clause=<compatible>	TYPE TO clause
  -fusage-type=<compatible>	USAGE type-name
  -fsynchronized-clause=<compatible>	SYNCHRONIZED clause
  -fspecial-names-clause=<compatible>	SPECIAL-NAMES clause
  -fgoto-statement-without-name=<compatible>	GOTO statement without name
  -fstop-literal-statement=<compatible>	STOP-literal statement
  -fstop-identifier-statement=<compatible>	STOP-identifier statement
  -fdebugging-mode=<compatible>	DEBUGGING MODE and debugging indicator
  -fuse-for-debugging=<compatible>	USE FOR DEBUGGING
  -fpadding-character-clause=<compatible>	PADDING CHARACTER clause
  -fnext-sentence-phrase=<compatible>	NEXT SENTENCE phrase
  -flisting-statements=<compatible>	listing-directive statements EJECT, SKIP1, SKIP2, SKIP3
  -ftitle-statement=<compatible>	listing-directive statement TITLE
  -fentry-statement=<compatible>	ENTRY statement
  -fmove-noninteger-to-alphanumeric=<compatible>	move noninteger to alphanumeric
  -fmove-figurative-constant-to-numeric=<compatible>	move figurative constants to numeric
  -fmove-figurative-space-to-numeric=<compatible>	move figurative constant SPACE to numeric
  -fmove-figurative-quote-to-numeric=<compatible>	move figurative constant QUOTE to numeric
  -fodo-without-to=<compatible>	OCCURS DEPENDING ON without to
  -fsection-segments=<compatible>	section segments
  -falter-statement=<compatible>	ALTER statement
  -fcall-overflow=<compatible>	OVERFLOW clause for CALL
  -fnumeric-boolean=<compatible>	boolean literals (B'1010')
  -fhexadecimal-boolean=<compatible>	hexadecimal-boolean literals (BX'A')
  -fnational-literals=<compatible>	national literals (N'UTF-16 string')
  -fhexadecimal-national-literals=<compatible>	hexadecimal-national literals (NX'265E')
  -fnational-character-literals=<compatible>	non-standard national literals (NC'UTF-16 string')
  -fhp-octal-literals=<compatible>	HP COBOL octal literals (%377)
  -facu-literals=<compatible>	ACUCOBOL-GT literals (#B #O #H #X)
  -fword-continuation=<compatible>	continuación de palabras COBOL
  -fnot-exception-before-exception=<compatible>	NOT ON EXCEPTION before ON EXCEPTION
  -faccept-display-extensions=<compatible>	extensions to ACCEPT and DISPLAY
  -frenames-uncommon-levels=<compatible>	RENOMBRA ítemes de 01-, 66- y 77-nivel
  -fsymbolic-constant=<compatible>	constants defined in SPECIAL-NAMES
  -fconstant-78=<compatible>	constant with level 78 item (note: has left to right precedence in expressions)
  -fconstant-01=<compatible>	constant with level 01 CONSTANT AS/FROM item
  -fperform-varying-without-by=<compatible>	PERFORM VARYING without BY phrase (implies BY 1)
  -freference-out-of-declaratives=<compatible>	references to sections not in DECLARATIVES from within DECLARATIVES
  -fprogram-prototypes=<compatible>	CALL/CANCEL con program-prototype-name
  -fcall-convention-mnemonic=<compatible>	specifying call-convention by mnemonic
  -fcall-convention-linkage=<compatible>	specifying call-convention by WITH ... LINKAGE
  -fnumeric-value-for-edited-item=<compatible>	numeric literals in VALUE clause of numeric-edited items
  -fincorrect-conf-sec-order=<compatible>	orden incorracta de parágrados CONFIGURACIÓN SECCIONAL
  -fdefine-constant-directive=<compatible>	allow >> DEFINE CONSTANT var AS literal
  -ffree-redefines-position=<compatible>	REDEFINES clause not following entry-name in definition
  -frecords-mismatch-record-clause=<compatible>	record sizes does not match RECORD clause
  -frecord-delimiter=<compatible>	RECORD DELIMITER clause
  -fsequential-delimiters=<compatible>	BINARY-SEQUENTIAL and LINE-SEQUENTIAL phrases in RECORD DELIMITER
  -frecord-delim-with-fixed-recs=<compatible>	RECORD DELIMITER clause on file with fixed-length records
  -fmissing-statement=<compatible>	missing statement (e.g. empty IF / PERFORM)
  -fzero-length-literals=<compatible>	zero-length literals, e.g. '' and ""
  -fxml-generate-extra-phrases=<compatible>	XML GENERATE's phrases other than COUNT IN
  -fcontinue-after=<compatible>	AFTER phrase in CONTINUE statement
  -fgoto-entry=<compatible>	ENTRY FOR GOTO and GOTO ENTRY statements
  -fassign-variable=<compatible>	ASSIGN [TO] variable in SELECT
  -fassign-using-variable=<compatible>	ASSIGN USING/VARYING variable in SELECT
  -fassign-ext-dyn=<compatible>	ASSIGN EXTERNAL/DYNAMIC in SELECT
  -fassign-disk-from=<compatible>	ASSIGN DISK FROM variable in SELECT
  -fvsam-status=<compatible>	VSAM status in FILE STATUS
	donde <compatible> es uno de lo siguiente:
	'ok', 'warning', 'archaic', 'obsolete', 'skip', 'ignore', 'error', 'unconformable'
  -fnot-reserved=<palabra>	palabra para ser tomada fuera de la lista de palabras reservadas
  -freserved=<palabra>   palabra a ser añadida para listado de palabras reservadas
  -freserved=<palabra>:<alias>	palabra a ser añadida para listado de palabras reservadas como alias
  -fnot-register=<palabra>	special register to disable
  -fregister=<palabra>   special register to enable
